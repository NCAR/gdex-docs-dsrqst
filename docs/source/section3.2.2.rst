3.2.2 - Set Request Information
=================================


.. _SR:

Action Option -**SR** (-**SetRequest**) :
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

creates and modifies data request information into GDEXDB
for a given dataset number and/or groups. One or multiple records can be
processed each time.

Although a new request record can be added by a specialist, it is rarely
needed. A request record should be generated by the Web interfaces that accept
submission of data requests. This Action is more commonly used to modify requests for
changing request status from Wait to Queue, or for changing purge date/time
for purging a request sooner or later.

.. _mode3.2.2:

:ref:`Mode option <section4>` that can be specified for setting request Action:

.. list-table::
   :widths: auto
   :header-rows: 0

   * - :ref:`-(GU|GetUsage) <GU>`
     - gets usage information of given requests and records into order metrics
   * - :ref:`-(MD|MyDataset) <MD>`
     - sets information into GDEXDB no matter the specialist who runs **dsrqst** owns the dataset or not
   * - :ref:`-(NR|NewRequest) <NR>`
     - sets a new request record into GDEXDB for given dataset, group index and request type

If information of a request exists already in GDEXDB for a given request index
or request name per option :ref:`-RI <RI>` (-RequestIndex) or :ref:`-RN <RN>` (-RequestName), the
request record is modified; otherwise, a new request record is added if
request index is 0 and :ref:`Mode option <section4>` :ref:`-NR <NR>` (-NewRequest) is present.

The request name is auto-generated, a combination of the user's uppercased
last name and the request index. It cannot be reset to anything else.

Default request processing priority is 10, although it can be set between 1
and 10, where 1 is the highest priority. A specialist can push a request to be
processed earlier by setting a higher priority per option :ref:`-PO <PO>` (-Priority),
in case many requests are queued for processing.


.. _3.2.2_e4:

**EXAMPLE 4. Set request status to 'Q' as queued for request index 42:**

| **dsrqst** :ref:`SR <SR>` :ref:`-RI <RI>` 42 :ref:`-RS <RS>` Q

A customized request can be created to stage existing data prepared for a specific
user.
